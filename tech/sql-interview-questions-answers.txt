<p><a class="accordion-trigger ques" id="ques19"><strong>Question 20. What Are The Different Types Of Joins?</strong></a></p>
<p><span class="ans"><strong>Answer :</strong></span><p><strong>Types of Joins are as follows:</strong></p>
<p><strong>INNER JOIN:</strong> It is also known as SIMPLE JOIN which returns all rows from BOTH tables when it has at least one column matched</p>
<p><strong>Syntax:</strong> SELECT column_name(s) </p>
<p>FROM table_name1 </p>
<p>INNER JOIN table_name2 </p>
<p>ON column_name 1=column_name 2;</p>
<p><strong>LEFT JOIN (LEFT OUTER JOIN):</strong> This join returns all rows from a LEFT table and its matched rows from a RIGHT table.</p>
<p><strong>Syntax:</strong> SELECT column_name(s)</p>
<p>FROM table_name1</p>
<p>LEFT JOIN table_name2</p>
<p>ON column_name 1=column_name 2;</p>
<p><strong>RIGHT JOIN (RIGHT OUTER JOIN):</strong> This joins returns all rows from the RIGHT table and its matched rows from a LEFT table.</p>
<p><strong>Syntax:</strong> SELECT column_name(s)</p>
<p>FROM table_name1</p>
<p>RIGHT JOIN table_name2</p>
<p>ON column_name1=column_name2;</p>
<p><strong>FULL JOIN (FULL OUTER JOIN):</strong> This joins returns all when there is a match either in the RIGHT table or in the LEFT table.</p>
<p><strong>Syntax:</strong> SELECT column_name(s)</p>
<p>FROM table_name1</p>
<p>FULL OUTER JOIN table_name2</p>
<p>ON column_name1=column_name2;</p></p>

<p><a class="accordion-trigger ques" id="ques25"><strong>Question 26. What Is Normalization?</strong></a></p>
<p><span class="ans"><strong>Answer :</strong></span><p>Normalization is the process of table design to minimize the data redundancy.</p>
<p><strong> There are different types of Noramalization forms in SQL:-</strong></p>
<ul>
<li><strong>First Normal Form (1NF):</strong> It removes all duplicate columns from the table. Creates table for related data and identifies unique column values</li>
<li><strong>First Normal Form (2NF):</strong> Follows 1NF and creates and places data subsets in an individual table and defines relationship between tables using primary key</li>
<li><strong>Third Normal Form (3NF):</strong> Follows 2NF and removes those columns which are not related through primary key</li>
<li><strong>Fourth Normal Form (4NF):</strong> Follows 3NF and do not define multi-valued dependencies. 4NF also known as BCNF</li>
</ul></p>

<p><span class="ans"><strong>Answer :</strong></span><p><strong>The difference between the Delete, Truncate and Drop command is:</strong></p>
<ul>
<li>Delete command is a DML command, it is used to delete rows from a table. It can be rolled back.</li>
<li>Truncate is a DDL command, it is used to delete all the rows from the table and free the space containing the table. It cant be rolled back.</li>
<li>Drop is a DDL command, it removes the complete data along with the table structure(unlike truncate command that removes only the rows). All the tables’ rows, indexes, and privileges will also be removed.</li>
</ul></p>

<p><strong>Syntax:</strong> REVOKE privilege_name</p>
