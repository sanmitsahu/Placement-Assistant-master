<p><span class="ans"><strong>Answer :</strong></span><p>Basically, the Document Object Model is a way of looking at a document. In that sense, it isn't all that different to CSS, which uses another kind of mental model. CSS uses the idea of selectors to interact with a document. The DOM, on the other hand, views a document as a tree of nodes: element nodes, attribute nodes and text nodes. The DOM provides a vocabulary for targeting and manipulating those nodes.<br/>If a programming language supports the DOM then you can use that same vocabulary to address any kind of markup document.</p>

<ol>
<li> Core DOM - standard model for any structured document</li>
<li> XML DOM - standard model for XML documents</li>
<li> HTML DOM - standard model for HTML documents</li>

<p>The DOM defines the objects and properties of all document elements, and the methods (interface) to access them.</p></p>
